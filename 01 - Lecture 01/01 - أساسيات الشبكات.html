📡 ملخصات CCNA & أساسيات الشبكات

📘 من إعداد مدرّع معتمد — نظام سيسكو (Cisco CCNA) — تبسيط احترافي للمفاهيم الأساسية

🔧 الأدوات المستخدمة (voice & Data)

1- Splitter: يقوم بتقسيم الحرارة إلى (voice & Data).

2- Filter: يعمل على تنقية الصوت.

3- Rosetta (RJ11): تعالج سلك التليفون.

4- Connector (RJ45): تعالج سلك الإنترنت.

📌 NIC (Network Interface Card): كارت تعريف الشبكة أو الإنترنت.

📌 MAC Address (Media Access Control address): عنوان جهازي على شبكة الإنترنت.

🌐 أنواع الإنترنت

1- Dial Up: أقدم أنواع الإنترنت، بطيء جداً، لا يقسم حرارة التليفون، مكلف جداً.
2- Broad band: باقات التليفون للموبايلات فقط بدون أسلاك، سريعة ولكن تتأثر بالجدران.
3- DSL: نوعان:
ADSL: Upload = 1/4 Download
SDSL: Upload = Download

🏢 أنواع الشبكات (LAN & WAN)

🌍 WAN (Wide Area Network)
شبكة خارجية متصلة بالإنترنت.
الأجهزة المستخدمة: Router, Modem (يقرؤون الـ IP ويخرجوننا لشبكة الإنترنت).
🏠 LAN (Local Area Network)
شبكة داخلية بدون الإنترنت.
الأجهزة: HUB, Switch, Router, Modem.

⚙️ المقارنات بين أجهزة الشبكة

HUB
16 مخرج، طريقة Broadcast (إرسال عشوائي). لا يمكن إرسال لجهاز محدد.
Switch
48 مخرج، طريقتين: Uni cast (واحد لواحد) & Multi cast (واحد لكثير).
Router
48 مخرج، يعمل بـ Routing table ويقرأ MAC Address و IP ويوجه البيانات.
Modem
4 مخارج، خاصية Modulation تحويل Analog (Voice/Data) ويقرأ MAC, IP.

📟 IP Address (Internet Protocol Address)

إصدارات الـ IP: IPv4 (4 octet) و IPv6 (8 octet). وظيفته: المسؤول عن الخروج على الشبكة الخارجية.

Classالنطاقالبتات الأولىSubnet mask (افتراضي)
Class A1.0.0.0 — 126.255.255.2550xxx255.0.0.0 (N.H.H.H)
Class B128.0.0.0 — 191.255.255.25510xx255.255.0.0 (N.N.H.H)
Class C192.0.0.0 — 223.255.255.255110x255.255.255.0 (N.N.N.H)
⚠️ ملاحظة: رقم 127 محجوز لـ Loopback (127.0.0.1) لاختبار حل مشاكل الشبكة.
Subnet mask: يحول الـ IP لأرقام لتوزيع الأجهزة؛ N ثابت=255، H متغير=0. الـ host يحمل حتى 255 جهاز.

🧠 بروتوكولات وخدمات الشبكة الأساسية

1- VOIP (Voice Over Internet Protocol): المسؤول عن الاتصال بالإنترنت عبر الأقمار الصناعية.

2- DNS (Domain Name System): تحويل اسم الموقع إلى IP.

3- URL (Uniform Resource Locator): ترتيب عنوان الموقع + بروتوكول Http.

4- DHCP (Dynamic Host Configuration Protocol): موجود في المودم، يوزع IP للأجهزة، حال العطل نعمل Rest/Configuration.

5- APIPA (Automatic Private IP Address): يحل محل DHCP ويعطي IP مميز 169.254.x.x أول رقمين ثابتين، يمنع تصادم IP.

6- NAT (Network Address Translation): مسؤول عن خروج الأجهزة من الداخلية للخارجية بتحويل IP الخاص (Private) إلى Real IP مختلف. أنواعه: Dynamic NAT, Static NAT, PAT.

7- PAT (Port Address Translation): تحويل Private IP إلى Real IP واحد على الشبكة.

مسميات الـ IP: Private IP (LAN)، Real IP (WAN/عام)، Public IP، Dynamic IP (متغير)، Static IP (ثابت).

🛠️ حل مشكلات الإنترنت (أشياء ملموسة وغير ملموسة)

📍 الأشياء الملموسة (لمبات)

✔ لمبة LAN (كابل) → غير مكان الكابل. ✔ لمبة WLAN (WiFi) خاصة بالواي فاي. ✔ لمبة Internet (Real IP) → نعمل Rest/Configuration. ✔ مشكلة DSL: نحدد 3 طرق: فصل سلك التليفون من Splitter وتوصيله بالمودم، إن اشتغل العيب في Splitter، وإبعاد السلك عن مجال كهربي.
إذا كانت لمبة DSL منورة ولكن يوجد ! أصفر أو X حمراء نتبع 6 خطوات: تغيير منفذ port، فحص كابل النت، فحص كارت الشبكة (Disable / Enable).

# أوامر تشخيص متقدمة:
netstat -a (لكشف الـ spyware, established أكثر من 4 مرات يدل على مشكلة)
nslookup google.com (لمعرفة IP الموقع)
ipconfig /all (كل جهاز متصل بالشبكة)
ipconfig /flushdns (مسح Cache DNS)
tracert google.com (تتبع المشكلة في الموقع)

✔ في tracert: إذا توقف قبل اسم الموقع → مشكلة مودم، إذا عند اسم الموقع → مشكلة سنترال، إذا تعدى الاسم وكمل → مشكلة الموقع نفسه أو تغيير الوقت/التاريخ.

📶 Wireless (مشاكل وحلول)

  • عدم الاتصال: التأكد من المودم وزر الواي فاي.
  • ازدحام الأجهزة: مسح بعض الشبكات.
  • الشبكة مخفية: كتابة الاسم يدويًا أو إظهارها.
  • أجهزة تقوية: Access Point (لمسافات بعيدة)، Repeater (لمسافات قريبة).
  • برنامج Proxy: اختراق المواقع المحجوبة. Fair wall: حماية الجهاز من الاختراق.

🕸️ أنواع الشبكات من حيث التوصيل (Topologies)

Bus
لا يمكن إرسال البيانات من الأجهزة ببعضها في نفس الوقت.
Ring
ترسل الملفات مثل عقارب الساعة، انقطاع وصلة يمنع الإرسال للجهاز المحدد.
Star
تلف الجهاز الرئيسي يسبب عدم الاتصال بالأجهزة.
Mesh
مكلفة جداً وكثيرة الأسلاك لأن كل الأجهزة متصلة ببعضها.

📚 الطبقات السبع (7 Layers of OSI)

الطبقةالاسمالمسؤولية / أمثلة
1Physicalأي شيء ملموس: HUB, NIC, Cable
2Data Linkحماية الـ Data، مثال: Switch
3Networkتنظيم الـ Data، مثال: Router
4Transportنقل البيانات عبر بروتوكولات (TCP, UDP)
5Sessionتكملة أي عملية على الإنترنت مثل Download
6Presentationتصنيف وعرض الملفات (MP3, JPG)
7Applicationمجموعة بروتوكولات: FTP (تحميل ورفع)، HTTP (تصفح)، SMTP (إرسال إيميلات)، POP (استقبال إيميلات)، TELNET (تحكم بجهاز آخر عن بعد).

📖 المصطلحات الهامة (CCNA)

TCP (Transmission Control Protocol): مسؤول عن إرسال واستقبال الـ Data.

UDP (User Datagram Protocol): مسؤول عن إرسال الـ Data فقط.

HTTP (Hyper Text Transfer Protocol): بروتوكول نقل النص التشعبي.

HTTPS (Hyper Text Transfer Protocol Security): بروتوكول نقل النص التشعبي الآمن.

FTP, SMTP, POP3, TELNET, DHCP, DNS, NAT, PAT كما تم شرحها أعلاه.

✅ خلاصة متقدمة لحل مشاكل الإنترنت

🔹 عند فحص الـ DSL: تغيير مكان الـ Port، تبديل كابل الإنترنت، مراجعة كارت الشبكة (enable/disable).
🔹 فحص سرعة الإنترنت عبر اختبار الـ Ms و %Lost: أقل من 20% مستقر، زيادة عن 45% بطيء.
🔹 استخدام أمر netstat -a لاكتشاف spyware (إغلاق المتصفحات، النظر إلى العمود الأيمن: أكثر من 4 مرات Established معناه ملفات تجسس → تنزيل Anti-spyware).
🔹 لمسح Cache DNS: ipconfig /flushdns لتسجيل مواقع جديدة.
🔹 في حال مشكلة المودم أو السنترال، نتبع tracert لتحديد موقع العطل بدقة.

📌 Loopback address: 127.0.0.1 لتشخيص الأجهزة العطلة على الشبكة.
Subnet mask يحدد أجزاء Network و Host: N ثابت = 255, H متغير = 0.
